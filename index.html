커넥션 풀 최적화와 성능 튜닝

📝 스터디 내용

데이터베이스 커넥션 풀(Connection Pool)의 개념과 필요성을 이해하고, HikariCP를 사용한 커넥션 풀 설정 튜닝 방법을 학습했습니다. 커넥션 생성 비용과 커넥션 풀의 자원 효율성을 중심으로, 스레드 풀과의 관계를 분석했습니다.

💡 배경 및 문제 인식

DB 연결 과정은 드라이버 로드, TCP 핸드쉐이크, 커넥션 생성 등으로 비용이 크며, MySQL 기준 INSERT 시에도 연결 관련 오버헤드가 상당합니다. 무작정 커넥션 수를 늘리는 것이 성능 개선의 답이 아니라는 점을 확인했습니다.

🔍 커넥션 풀 개념 및 장점

  • DB 커넥션을 미리 만들어 재사용하는 기술로, 매번 새로 만들지 않고 풀에서 꺼내 씀으로써 성능 향상
  • 서버 부하 감소 및 자원 효율성 증대
  • 하지만 커넥션 풀 크기만 키운다고 해서 성능이 좋아지지 않으며, 스레드 수와 균형을 맞춰야 함

🛠 실험: 병렬 요청 30,000건 처리 테스트

📈 해석 및 결론

  • 최대 풀 크기 10~20 이상부터 성능 향상이 뚜렷하게 나타남
  • 대기 커넥션 수는 최대 풀 크기보다 적당히 유지하는 것이 효율적
  • 스레드 풀을 반드시 설정하고, 스레드 수에 따라 성능 차이가 크게 발생

따라서 커넥션 풀과 스레드 풀의 적절한 조합을 통해 대량 병렬 처리 시 처리 시간을 크게 줄이고, 자원 낭비를 막는 것이 핵심임을 확인했습니다.
