Oship 해외배송 플랫폼

📝 프로젝트 설명

Oship은 해외 전자상거래 판매자(셀러)를 위한 국제 특송 요금 비교 및 배송신청 플랫폼입니다. 셀러는 DHL, FedEx, EMS 등 다양한 배송사의 실시간 요금을 비교하고, 가장 적합한 옵션을 선택해 해외배송을 신청할 수 있습니다. 플랫폼은 셀러, 포워딩회사, 배송사 간의 역할을 명확히 구분하여 각 주체가 본인의 영역에 집중할 수 있도록 설계되었습니다.

🛠 기술 스택

Spring Boot Redis MySQL Docker AWS Grafana Loki Prometheus

🔧 개발 사항

  • 배송 상태를 주기적으로 조회하는 스케줄러 개발 및 API 연동
  • JWT 인증 + Redis 기반 토큰 블랙리스트로 인증 구조 설계
  • URL 기반 권한 제어 및 CORS 적용
  • 퍼블릭/프라이빗 서브넷 분리, NAT Gateway 및 ALB 활용한 AWS 인프라 구성
  • GitHub Actions 기반 CI/CD 파이프라인, Bastion 서버 통한 자동 배포
  • Testcontainers 기반 통합 테스트 환경 구성 (MySQL, Redis)
  • Promtail + Loki 기반 로그 수집 및 Spring, Redis, MySQL 실시간 모니터링 구축
  • Excel 대량 업로드 시 CPU 과부하 탐지 및 최적화

💡 개발 관련 설명

상황

- 운영자 수작업으로 배송 상태를 수시로 조회
- Excel 업로드 시 CPU 부하 99%
- 부하테스트시 데이터베이스 커넥션 에러 발생
- 장애 발생 시 원인 파악이 어려움
- 테스트시 외부 인프라 의존도가 높아 CI 환경에서 테스트 어려움

해결

- 스케줄러를 통해 배송 상태를 자동 조회 및 갱신하도록 개선
- Promtail + Loki + Grafana로 로그를 수집해 API 오류를 시각화
- grafana 시각화를 통해 Spring, Redis, MySQL 상황 실시간 파악
- HikariCP 커넥션 수 조절하여 커넥션 에러 해결
- Excel 업로드시 배치처리 및 병렬처리 비교 후 배치처리 도입

결과

- 배송 상태 자동 추적으로 운영 편의성 및 정확도 향상
- 평균 CPU 사용률 99% → 80%로 감소
- API 장애 원인 조기 파악 가능해져 운영 안정성 확보
- Testcontainers 기반 테스트 환경으로 실운영 환경과 유사한 테스트 가능
- CI/CD 파이프라인 구축으로 배포 시간 단축 및 오류율 감소
